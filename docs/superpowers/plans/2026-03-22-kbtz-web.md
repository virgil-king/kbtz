# kbtz-web Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a web server (`kbtz-web`) that provides browser-based access to the kbtz workspace with chat-style agent session views and a live task tree.

**Architecture:** Single-process axum server managing detached JSON shepherd processes. Reuses lifecycle logic from a newly extracted `kbtz-workspace-core` crate. Solid (SolidJS) frontend communicates via WebSocket.

**Tech Stack:** Rust (axum, tokio, rusqlite), SolidJS, WebSocket, Unix sockets

**Spec:** `docs/superpowers/specs/2026-03-22-kbtz-web-design.md`

/// Write state file atomically (temp + rename).
pub fn write_state_file(path: &Path, state: &ShepherdState) -> Result<()> {
    let tmp = path.with_extension("tmp");
    std::fs::write(&tmp, serde_json::to_string_pretty(state)?)?;
    std::fs::rename(&tmp, path)?;
    Ok(())
}

    let content = std::fs::read_to_string(path)?;

- [ ] **Step 4: Implement the shepherd main function**

Structure the binary with these responsibilities:
1. Parse CLI args: `--socket-path`, `--state-file`, `--history-limit`, `--readiness-fd`, then the agent command + args
2. Spawn the agent process with stdin/stdout pipes (NOT a PTY — the agent speaks JSON)
3. Write the atomic state file (JSON with `session_id`, `shepherd_pid`, `child_pid`, `socket_path`, `task`, `agent_type`)
4. Write the socket path to the readiness pipe fd, close it
5. Listen on the Unix socket (single client at a time)
6. Main loop: read JSON events from agent stdout → push to ring buffer + forward to connected client. Read `Input` messages from client → write to agent stdin. Handle `Shutdown` → send SIGTERM to agent.
7. On exit: remove state file, remove socket file

Reference `kbtz-workspace/src/bin/kbtz-shepherd.rs` for the socket handling pattern, but this is simpler — no VTE, no resize, no PTY.

/// Authenticator checks an HTTP request for valid credentials.
/// The spec defines this at the request level (checking cookies) rather
/// than the token level, so future OAuth can inspect headers/cookies directly.
pub trait Authenticator: Send + Sync {
    fn authenticate(&self, request: &axum::http::Request<axum::body::Body>) -> Result<(), AuthError>;
}

#[derive(Debug)]

pub fn load_or_create_token(path: &Path) -> Result<String> {
    if path.exists() {
        Ok(std::fs::read_to_string(path)?.trim().to_string())
    } else {
        let token: String = (0..32)
            .map(|_| format!("{:02x}", rand::random::<u8>()))
            .collect();
        std::fs::write(path, &token)?;
        // Set 0600 permissions
        #[cfg(unix)]
        {
            use std::os::unix::fs::PermissionsExt;
            std::fs::set_permissions(path, std::fs::Permissions::from_mode(0o600))?;
        }
        Ok(token)
    }
}

## Task 12: WebSocket Client

**Files:**
- Create: `kbtz-web-ui/src/lib/ws.ts`

- [ ] **Step 1: Implement WebSocket client with reconnection**

```typescript
import { createSignal } from "solid-js";
import type { ServerMessage, ClientMessage } from "./types";

export function createWebSocket(url: string) {
  const [connected, setConnected] = createSignal(false);
  const handlers = new Map<string, Set<(msg: ServerMessage) => void>>();
  let ws: WebSocket | null = null;

  function connect() {
    ws = new WebSocket(url);
    ws.onopen = () => setConnected(true);
    ws.onclose = () => {
      setConnected(false);
      setTimeout(connect, 2000); // reconnect after 2s
    };
    ws.onmessage = (e) => {
      const msg: ServerMessage = JSON.parse(e.data);
      handlers.get(msg.type)?.forEach(fn => fn(msg));
    };
  }

  function send(msg: ClientMessage) {
    ws?.send(JSON.stringify(msg));
  }

  function on(type: string, handler: (msg: ServerMessage) => void) {
    if (!handlers.has(type)) handlers.set(type, new Set());
    handlers.get(type)!.add(handler);
    return () => handlers.get(type)?.delete(handler);
  }

  connect();
  return { connected, send, on };
}
```

import type { AgentEvent } from "./types";
import { JSX } from "solid-js";

type Renderer = (event: AgentEvent) => JSX.Element;

const renderers: Record<string, Renderer> = {};

export function registerRenderer(type: string, renderer: Renderer) {
  renderers[type] = renderer;
}

export function renderEvent(event: AgentEvent): JSX.Element {
  const renderer = renderers[event.type];
  if (renderer) return renderer(event);
  // Fallback: monospace raw JSON
  return <pre style="font-size: 13px; white-space: pre-wrap;">{JSON.stringify(event, null, 2)}</pre>;
}

// Register default text renderers
registerRenderer("assistant", (e) => <div class="message assistant">{e.message?.content || e.text || JSON.stringify(e)}</div>);
registerRenderer("tool_use", (e) => <div class="message tool-use"><strong>{e.name}</strong>: <code>{JSON.stringify(e.input)}</code></div>);
registerRenderer("tool_result", (e) => <div class="message tool-result"><pre>{typeof e.content === "string" ? e.content : JSON.stringify(e.content)}</pre></div>);
